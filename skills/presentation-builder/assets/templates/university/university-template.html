Key concept

What is a graph neural network?

Definition

Graph Neural Network

fθ : 𝒢 → ℝd

A neural network whose forward pass operates directly on a graph G = (V, E). Its output is invariant to vertex relabelling — the model sees the same molecule no matter which atom is numbered first.

In practice
  • Permutation-invariant by construction. Reordering atoms does not change the prediction; symmetry is baked into the architecture.
  • Local message passing, then global readout. Each node first mixes information with its neighbours, then a pooling step collapses the graph into a single vector.
  • End-to-end trainable. Gradients flow from the prediction back to the atomic embeddings; no hand-engineered features in the middle.

01 — Background

Problem framing

Three properties of molecular data make it a poor fit for standard sequence and vector models.

  • Variable, irregular structure. A molecule has no canonical ordering of atoms. Any model that imposes one inherits a symmetry it doesn't have.
  • Locality with long-range effects. Most chemistry is local — bonds and immediate neighbours — but a handful of properties depend on whole-molecule context.
  • Small, noisy labelled datasets. Wet-lab measurements are expensive. Most public benchmarks are in the 1k–40k range, with non-trivial label noise.

02 — Method

Related approaches

A · Fingerprint baseline

Random forests on Morgan fingerprints.

Hand-engineered substructure counts feed a tree ensemble. Fast, interpretable, surprisingly hard to beat on small datasets.

  • Trains in seconds; no GPU required.
  • Fixed feature set — cannot specialise.
  • Strong floor on classification tasks.
B · Graph neural network

Message passing over the molecular graph.

Node and edge embeddings are learned end-to-end from labels. Representations adapt to the target property.

  • Permutation invariant by construction.
  • Scales to multi-task & pretraining setups.
  • Needs > 5k examples to clearly separate.

02 — Method

Three families of graph neural networks

A · Spectral

Filters in the Laplacian eigenbasis.

ChebNet, GCN, and friends. Theoretically clean; expensive to transfer across graphs of different size.

  • Strong theoretical grounding.
  • Cost scales with graph size.
  • Mostly historical in chemistry.
B · Spatial (message passing)

Aggregates from each node's local neighbourhood.

MPNN, GINE, GraphSAGE. The mainstream choice for molecular tasks today — strong inductive bias matched to chemistry.

  • Permutation-invariant by construction.
  • Best accuracy / cost for ≤ 100-atom graphs.
  • What our experiments compare.
C · Attention-based

Each node learns how to weight its neighbours.

GAT, Graph Transformers (GraphGPS). Heavier per step; payoff is task-dependent.

  • Captures heterogeneous neighbours.
  • Quadratic in neighbourhood size.
  • Promising for large-graph regimes.

02 — Method

The message-passing update

hv(l+1) = σ ( W(l) · MEANu ∈ 𝒩(v) ∪ {v} ( hu(l) ) )

Applied l = 0 … L−1 times to produce the final node embeddings.

hv(l)
Embedding of node v at layer l.
𝒩(v)
One-hop neighbourhood of v.
W(l)
Learnable weights at layer l.
σ
Non-linearity (we use ReLU).
MEAN
Permutation-invariant aggregator.
L
Total number of layers (L = 4).

Method · Pseudocode

02 — Method

Forward pass

forward_pass.py 48 lines · python
1def forward(self, batch):
2# 1. atom & bond features → embeddings
3h = self.atom_embed(batch.x)
4e = self.bond_embed(batch.edge_attr)
6# 2. L rounds of message passing
7for layer in self.layers:
8h_new = layer(h, batch.edge_index, e)
9h = self.norm(h + h_new) # residual
11# 3. set2set readout → graph vector
12g = self.readout(h, batch.batch)
14# 4. MLP head → property
15return self.head(g)

Initial embeddings

Each atom is mapped to a learned 32-d vector from atomic number, charge, hybridisation, aromaticity.

Message rounds

Four GINE-edge layers. Residual + LayerNorm keep the deeper stack stable during training.

Readout & predict

Set2Set pools per-node embeddings into one graph vector; a two-layer MLP head outputs the property.

Discussion · Contributions

04 — Discussion

Contributions

What this work adds — three claims, each backed by an ablation.

  1. 01

    Edge-feature augmentation gives a consistent +6% on small datasets.

    Bond type, stereochemistry, and ring membership added as 8-d edge features outperform vanilla GINE on BBBP, ClinTox, BACE.

    → Ablation: §4.2
  2. 02

    Set2Set readout beats sum pooling on every benchmark.

    Mean improvement +2.1 points; gap widens on datasets where molecule size varies more across examples.

    → Ablation: §4.3
  3. 03

    3D coordinates are unnecessary when edge features are rich enough.

    Adding atomic coordinates as input gives less than 0.4-point lift and adds substantial preprocessing cost.

    → Ablation: §4.4

04 — Discussion

Limitations & threats to validity

  • !

    One scaffold split per dataset.

    Scaffold splits are sensitive to which scaffolds end up in train vs. test.

    What we did Repeated across 5 seeds and 2 alternative splits in the ablation.
  • !

    Only classification benchmarks.

    Results may not transfer to regression targets like solubility or binding affinity.

    Next Regression analogue (FreeSolv, ESOL) planned for the thesis.
  • !

    No external validation set.

    MoleculeNet has been the field benchmark since 2018 — overfitting to it is plausible.

    Mitigation Held-out 2024 ChEMBL release planned for September.
